#### **Monday**

**Lab 18 — Enable Security Hub**

**Concepts:**

* AWS Security Hub is a cloud security posture management service that acts as a "command center," aggregating, organizing, and prioritizing security alerts (findings) from multiple AWS services into a single dashboard.

**Lab objectives:**

* Set up centralized security monitoring with AWS Security Hub.
* Enable automated security check standards.
* Assess the security compliance level of AWS resources.
* Become familiar with detecting security issues.

**Benefits**

* **Enterprise:**
    * Enhanced security monitoring capabilities.
    * Early detection of misconfigurations and reduced data breach risk.
    * Lower incident response costs.

**Lab steps:**

* **Step 1:** Access AWS Security Hub from the AWS Management Console and begin activation.
* **Step 2:** Confirm access permissions so Security Hub can collect security data from related AWS services.
* **Step 3:** Enable the AWS Foundational Security Best Practices standard.
* **Step 4:** Confirm configuration and complete Security Hub activation.
* **Step 5:** Review the Dashboard and Findings to assess the security compliance status of AWS resources.

#### **Tuesday & Wednesday & Thursday & Friday**

**Lab 117 — Build a Complete serverless Chat Website**

**Service concepts:**

* **Amazon S3:** Stores and distributes static application files.
* **AWS Lambda:** Handles application business logic in a serverless model.
* **Amazon API Gateway:** Receives user requests and connects to Lambda.
* **Amazon DynamoDB:** Stores application data with high performance.
* **Amazon Cognito:** Manages user registration, sign-in, and authentication.
* **Amazon CloudFront:** Accelerates content delivery through a CDN network.
* **AWS IAM:** Manages access to AWS resources following the principle of least privilege.

**Benefits:**

* **Enterprise:**
    * Cost savings.
    * Auto scaling.
    * High security.
    * Fast deployment.
    * Good performance.
    * Reduced administrative burden.

**Lab objectives:**

* Build a serverless web application on AWS.
* Deploy the website using S3, Lambda, and API Gateway.
* Store data with DynamoDB.
* Authenticate users with Cognito.
* Improve performance with CloudFront.
* Manage and clean up AWS resources.
